# Contributing to Kettle

Welcome to **Kettle** and thank you for your interest in contributing to the project! We’re excited to have you here. Every contribution code or non code helps improve the project and grow the community.

This document outlines how you can get involved, set up your environment, and make meaningful contributions to the **pointblank-club/kettle** repository.

---

## Code of Conduct

Kettle follows the spirit of open-source collaboration and mutual respect. All contributors are expected to:

* Be respectful and inclusive
* Welcome diverse perspectives
* Provide constructive feedback
* Act professionally in all community interactions

By participating in this project, you agree to uphold these principles.

---

## Getting Started

If you are new to the project or to open source in general, here’s how to begin:

1. **Explore the repository**

   * Read the README to understand the project goals and architecture.
   * Browse existing issues and pull requests to see what’s being worked on.

2. **Fork & Clone**

   * Fork the repository to your GitHub account.
   * Clone your fork locally:

     ```bash
     git clone https://github.com/<your-username>/kettle.git
     cd kettle
     ```

3. **Create a Branch**

   ```bash
   git checkout -b feature/your-feature-name
   ```

## Ways to Contribute

Contributions are not limited to writing code. We welcome improvements across the project.

### Code Contributions

You can contribute by:

* Implementing new features or enhancements
* Fixing bugs or performance issues
* Improving security or reliability
* Refactoring existing code without changing behavior
* Adding or improving unit, integration, or end-to-end tests

**Guidelines:**

* Keep changes focused and scoped
* Follow existing code style and conventions
* Add comments where logic is non-obvious
* Ensure new functionality is covered by tests

---

### Non-Code Contributions

Non-code contributions are equally valuable:

* **Documentation**: Fix typos, improve clarity, add examples, or write guides
* **Issue Triage**: Help label, reproduce, or clarify issues
* **Design & UX**: Suggest improvements to APIs, workflows, or developer experience
* **Community Support**: Answer questions, help newcomers, and share best practices
* **Content Creation**: Blogs, tutorials, demos, or presentations related to Kettle

---

## Issues & Labels

We maintain a curated issue list to help contributors find suitable tasks:

* **Good First Issue** : Beginner-friendly tasks
* **Help Wanted** : Requires additional input or effort
* **Backlog** : Planned or pending work suitable for deeper contributions

If you plan to work on an issue, please comment on it to avoid duplicate effort.

---

## Pull Request Process

1. Ensure your branch is up to date with `main`
2. Push your changes to your fork
3. Open a pull request against `main`
4. Clearly describe:

   * What the change does
   * Why it’s needed
   * Any relevant issues it addresses (e.g., `Fixes #123`)

### PR Best Practices

* Keep PRs small and focused
* Write clear commit messages
* Respond to review feedback promptly and respectfully
* Be open to suggestions and improvements

---

## Community & Support

If you need help or want to engage with the community:

* Use **GitHub Issues** for bugs and feature requests
* Use **GitHub Discussions** for ideas and long-form questions
* Participate in project discussions and reviews
* Help others when you can, mentorship is encouraged

---

## Growth & Ownership

Consistent, high quality contributors may take on larger roles over time, including:

* Reviewing pull requests
* Helping guide technical decisions
* Mentoring new contributors
* Shaping the project roadmap

We value initiative, collaboration, and long-term commitment.
